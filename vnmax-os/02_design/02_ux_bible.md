# Prompt 07 - VNMAX UX Bible

## Objetivo

Toda interface criada para a VNMAX deve reduzir esforco cognitivo, aumentar previsibilidade e transmitir inteligencia operacional.

## Filosofia

Interfaces nao existem para impressionar. Existem para resolver problemas. Uma interface bonita, mas dificil, e um fracasso. Uma interface simples, rapida e clara e um sucesso.

## Regra maxima

Toda tela deve responder imediatamente:

1. Onde estou?
2. O que posso fazer?
3. Como faco?
4. O que acontece depois?

Se qualquer resposta exigir esforco mental, o fluxo precisa ser redesenhado.

## Principios obrigatorios

- Clareza.
- Simplicidade.
- Consistencia.
- Rapidez.
- Precisao.
- Feedback.
- Escalabilidade.
- Acessibilidade.
- Performance.
- Confiabilidade.

## Carga cognitiva

- Minimize escolhas.
- Agrupe funcionalidades.
- Mostre primeiro o essencial.
- Evite menus gigantes.
- Favoreca reconhecimento em vez de memorizacao.

## Leis de UX

### Hick

Quanto mais opcoes, maior o tempo de decisao. Reduza menus, agrupe acoes e mostre apenas o necessario.

### Fitts

Botoes importantes devem ser grandes e acessiveis. Acoes destrutivas devem ficar separadas de acoes como salvar.

### Miller

Agrupe informacoes em blocos pequenos. Evite listas enormes sem filtros ou busca.

### Jakob

Usuarios esperam padroes familiares. Nao reinvente login, menu, busca, tabela, dashboard, configuracao, perfil ou notificacoes.

### Tesler

A complexidade deve ficar no software, nao no usuario.

## Padroes por superficie

### Landing pages

Ordem recomendada: Hero, Problema, Solucao, Beneficios, Servicos, Diferenciais, Prova Social, FAQ, CTA e Footer.

### Dashboards

Comecar por KPIs, depois graficos, depois tabelas, depois configuracoes.

### Formularios

Validacao em tempo real, erros claros, etapas curtas e preservacao dos dados preenchidos.

### Produtos de IA

Devem responder rapido, usar streaming quando possivel, permitir copiar, editar, continuar, interromper e regenerar.

### CRM

Pipeline visual, status claros, busca global, filtros rapidos e cards objetivos.

### ERP

Poucos cliques, eficiencia por teclado, consistencia e performance.

## Estados obrigatorios

- Loading.
- Empty.
- Success.
- Warning.
- Error.
- Offline.

Nunca deixar o usuario diante de uma tela vazia sem orientacao.

## Erros

- Nunca culpar o usuario.
- Nunca mostrar stack trace.
- Explicar o problema em linguagem humana.
- Oferecer recuperacao.

## Checklist final

Um usuario novo conseguiria usar esta interface sem treinamento? Se a resposta for nao, redesenhe.
